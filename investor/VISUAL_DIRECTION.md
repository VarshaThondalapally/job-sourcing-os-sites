# Visual Direction

## Design Target

Premium AI-native investor pitch website.

The site should feel like a new category:

```text
opportunity routing intelligence
```

It should not feel like:

- a resume portfolio
- a job board
- a generic SaaS landing page
- a chatbot demo
- a dark dashboard template
- a stock-photo career site

## Visual Principles

1. Architecture is the centerpiece.
2. The product feels like an operating system, not a content site.
3. The interface should show routing, verification, evidence, and decision gates.
4. Use cinematic depth through structure, motion, and information density, not decorative blobs.
5. Avoid fake logos, fake dashboards, fake customers, and fake metrics.
6. Use TODO labels where proof is missing.
7. Make failure audits feel like product-learning loops.

## Palette

Use a disciplined, high-contrast palette:

- near-black operational background
- ivory text and paper sections
- signal green for verified proof
- amber for warning / TODO / human approval
- blue for source flow
- red-orange for blocker or failure audit signals

Avoid:

- purple-blue gradient dominance
- generic glassmorphism
- decorative orbs
- beige-only startup decks
- fake neon cyberpunk styling

## Typography

Use strong system typography:

- large, compressed-feeling hero scale
- tight section titles
- small uppercase signal labels
- crisp product UI cards

Do not use negative letter spacing.

## Major Section Visual Concepts

### Hero

Visual concept:

A cinematic opportunity-routing map, not a person at a laptop.

The current site uses a canvas-based signal map:

- source nodes
- verification node
- route node
- fit node
- human node
- learning loop

If generating a hyper-realistic image, use this prompt:

```text
Hyper-realistic cinematic command center for an AI opportunity routing system, large wall-sized interface showing verified hiring routes, evidence trails, route gates, human approval checkpoints, and response feedback loops, no people, no robots, no fake company logos, premium investor-grade lighting, tactile glass and paper interface elements, realistic workroom environment, sharp details, high contrast, editorial technology photography, 35mm lens, believable product visualization
```

Negative prompt:

```text
robots, humanoid AI, stock photo recruiter, smiling office people, fake logos, crypto charts, purple gradient blobs, fantasy UI, illegible tiny text, generic dashboard, resume template
```

### Problem Section

Visual concept:

Split-screen contrast between shallow extraction and real hiring requirements.

Prompt:

```text
Realistic split-screen product visualization, left side showing fragmented job postings, stale mirrors, duplicated ATS links, and conflicting timestamps, right side showing verified route intelligence, human contact path, blocker checks, and response status, premium editorial lighting, no people, no logos, crisp paper-and-glass interface, believable enterprise software scene
```

### Core Insight

Visual concept:

The equation should feel like an operating thesis:

```text
active role + person-proven route + specific proof fit + tracked follow-up
```

Prompt:

```text
High-end product thesis visualization, four physical interface tiles on a dark work surface labeled active role, person-proven route, proof fit, tracked follow-up, thin illuminated routing lines connecting them into a closed operating loop, realistic materials, no people, no robots, premium technology editorial style
```

### Interactive 15-Layer Architecture

Visual concept:

Clickable pyramid or stack with each layer behaving like a system module.

Prompt:

```text
Hyper-realistic modular operating system architecture, fifteen stacked layers as physical glass-and-paper interface slabs, each layer has evidence labels, route gates, and human approval checkpoints, premium dark command environment, subtle signal lighting, no fake logos, no people, no robots, investor pitch quality
```

### Workflow Demo

Visual concept:

One opportunity moving through the system as a signal packet:

discover -> verify -> route -> prove -> package -> learn.

Prompt:

```text
Realistic workflow visualization of a single opportunity signal moving through verification gates, route intelligence, fit proof, blocker checks, application packet, outreach CRM, and response monitor, cinematic product photography, tactile UI panels, no human faces, no fake companies, crisp legible interface zones
```

### Failure Audits

Visual concept:

Incident-review wall, like a serious product postmortem system.

Prompt:

```text
Premium product failure audit wall, four incident cards labeled route intelligence miss, fresh-feed coverage gap, duplicate prevention, response monitor loop closure, evidence tags and correction arrows, realistic startup war-room setting, no people, no logos, no blame language, professional product learning atmosphere
```

### Human-In-The-Loop Safety

Visual concept:

AI prepares work, human approval gates decide sensitive actions.

Prompt:

```text
Realistic AI safety control interface for job sourcing operations, left side shows AI-prepared research, extracted facts, draft fit arguments, and blocker flags, right side shows locked human approval gates for sending emails, relocation stance, sensitive claims, and final fit sentence, premium enterprise product design, no robots, no people, no fake logos
```

### Investor Memo

Visual concept:

Clean investment memo surface with thesis, wedge, risks, milestones, and TODO metrics visibly marked.

Prompt:

```text
Realistic investor memo interface for an AI startup, structured sections for thesis, wedge, why now, product surface, defensibility, risks, next milestones, TODO metrics clearly marked, premium editorial product shot, crisp typography, no fake traction, no logos, no people
```

## Motion Direction

Use restrained motion:

- signal particles moving through the route map
- clickable architecture layer changes
- workflow step transitions
- pitch mode modal

Avoid:

- floating orbs
- confetti
- spinning 3D objects
- constant large animations
- distracting parallax

## Investor-Grade Checklist

Before sending:

- first screen must not look like a resume/job-search tool
- architecture must be the emotional centerpiece
- missing metrics must remain TODO
- failure audits must sound like product learning, not excuses
- AI boundary must be visible
- "job extraction is only Layer 3" must be clearly present
- no fake traction or logos
- no generic "AI helps you apply faster" framing
